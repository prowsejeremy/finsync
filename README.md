# Finsync

A native Android app that authenticates with a [Jellyfin](https://jellyfin.org) media server and syncs your music library to local device storage.

---

## 🚨 DISCLAIMER

A decent portion of this apps code has been generated by Claude and Gemini, while I have tested it pretty thoroughly, and made a fair few tweaks to the code myself it is important to note the LLMs contribution to this project.

---

## Features

- **Secure Authentication** — Login via Jellyfin's API; credentials stored in `EncryptedSharedPreferences` with AES-256.
- **Full Library Sync** — Downloads every audio item organised into `Artist/Album/Track` folders.
- **Incremental Sync** — Skips unchanged files tracked via a local Room database; only new or updated tracks are downloaded.
- **Album Art** — Cover art downloaded alongside each album.
- **Foreground Service** — Sync runs persistently with a progress notification.
- **Scheduled Sync** — WorkManager-based periodic sync (default: 6 hours), survives reboots.
- **Library Browser** — Browse server albums directly in-app.
- **Settings** — Configurable sync interval and storage directory from within the app.

## Architecture

```
app/src/main/java/com/jpd/finsync/
├── api/
│   ├── JellyfinApi.kt          Retrofit service interface
│   └── JellyfinClient.kt       OkHttp + Retrofit factory
├── auth/
│   ├── CredentialStore.kt      EncryptedSharedPreferences wrapper
│   └── JellyfinRepository.kt   Repository layer (login, items, albums)
├── db/
│   └── (Room entities, DAOs)   SyncedTrack, SyncedAlbum, SyncedArtwork
├── model/
│   └── Models.kt               Data classes (MediaItem, ServerConfig, SyncState…)
├── sync/
│   └── SyncEngine.kt           Core logic: enumerate → download → cleanup
├── service/
│   ├── SyncService.kt          Foreground service wrapper
│   ├── BootReceiver.kt         Re-schedules sync after device reboot
│   └── SyncWorker.kt           WorkManager periodic job
└── ui/
    ├── PermissionsActivity     Launcher — permission onboarding
    ├── LoginActivity           Server URL + credential entry
    ├── MainActivity            Sync control and status
    ├── LibraryActivity         Browse downloaded albums
    └── SettingsActivity        Sync interval and directory preferences
```

## Setup

1. Install the APK on your Android device.
2. Open the app and grant the requested permissions.
3. Enter your **Jellyfin server URL** (e.g. `https://jellyfin.home.arpa` or `http://192.168.1.10:8096`), **username**, and **password**, then tap **Sign In**.
4. On the main screen, tap **Sync Now**. Progress appears in the notification drawer.
5. To adjust the sync schedule or storage location, open the three-dot menu → **Settings**.

### Default sync directory

```
/sdcard/Music/Jellyfin/<ServerName>/
```

## Jellyfin API Endpoints

| Endpoint                                                      | Purpose                    |
| ------------------------------------------------------------- | -------------------------- |
| `GET /System/Info/Public`                                     | Verify server reachability |
| `POST /Users/AuthenticateByName`                              | Login                      |
| `GET /Users/{id}/Items?IncludeItemTypes=Audio&Recursive=true` | Enumerate tracks           |
| `GET /Users/{id}/Items?IncludeItemTypes=MusicAlbum`           | Browse albums              |
| `GET /Audio/{id}/file`                                        | Download audio             |
| `GET /Items/{id}/Images/Primary`                              | Download album art         |

## Permissions

| Permission                                            | Purpose                                |
| ----------------------------------------------------- | -------------------------------------- |
| `INTERNET`                                            | Connect to Jellyfin server             |
| `READ_MEDIA_AUDIO` / `WRITE_EXTERNAL_STORAGE`         | Read/write music files                 |
| `MANAGE_EXTERNAL_STORAGE`                             | Broad storage access (API 11+)         |
| `FOREGROUND_SERVICE` + `FOREGROUND_SERVICE_DATA_SYNC` | Background sync                        |
| `POST_NOTIFICATIONS`                                  | Sync progress notification             |
| `RECEIVE_BOOT_COMPLETED`                              | Re-schedule sync after reboot          |
| `WAKE_LOCK`                                           | Keep CPU awake during WorkManager jobs |

## Known Limitations

- **No transcoding** — Files download in their original format.
- **No in-app playback** — Playback is handled by your device's music player after sync.
- **Large libraries** — A 50,000-track library will take significant time and storage on first sync; subsequent syncs are incremental.
